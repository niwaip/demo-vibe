# VK-Plan: 项目规划与拆解技能

**独立技能，无需任何 MCP 或外部依赖**

---

## 技能概述

| 属性 | 值 |
|------|-----|
| 名称 | VK-Plan |
| 依赖 | 无（纯方法论） |
| 触发 | 用户提出新需求 |
| 输出 | 功能点清单、任务列表、OpenSpec 规格 |
| 下一步 | 确认后调用 VK-Execute |

---

## 输入

用户需求描述（可以是模糊的、不完整的）

---

## 输出

1. **功能点清单**：结构化的功能分解
2. **任务列表**：原子化开发任务
3. **依赖图**：任务间依赖关系
4. **批次划分**：并行执行策略
5. **文件所有权矩阵**：避免冲突的文件分配
6. **OpenSpec 规格**：每个任务的详细规格文档

---

## 执行流程

### Phase 1: REFINE（需求细化）

**目标**：将模糊需求转化为清晰的功能点

#### 步骤

1. **澄清需求**（必要时向用户提问）：
   - 这个功能解决什么问题？
   - 用户是谁？使用场景是什么？
   - 输入是什么？输出是什么？
   - 有什么边界条件/异常情况？
   - 有什么性能/安全要求？

2. **输出功能点清单**：

```markdown
## 功能点清单

| ID | 功能点 | 优先级 | 复杂度 | 依赖 | 说明 |
|----|--------|--------|--------|------|------|
| F1 | {名称} | P0/P1/P2 | 低/中/高 | - | {描述} |
| F2 | {名称} | P0 | 中 | F1 | {描述} |
| ... | ... | ... | ... | ... | ... |

优先级说明：
- P0: 必须实现（核心功能）
- P1: 应该实现（重要功能）
- P2: 可以实现（可选功能）

复杂度说明：
- 低: < 2小时
- 中: 2-4小时
- 高: > 4小时
```

---

### Phase 2: SPLIT（任务划分）

**目标**：将功能点拆解为可独立开发的原子任务

#### 拆解原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 每个任务只做一件事 |
| 文件隔离 | 每个任务有独立的文件所有权 |
| 可测试性 | 每个任务有独立的测试用例 |
| 粒度适中 | 预计工时 0.5-4 小时 |

#### 步骤

1. **拆解功能点为任务**

2. **分析依赖关系**：
   - 必须先行：A 必须在 B 之前完成
   - 可以并行：A 和 B 无依赖
   - 必须后置：A 必须在 B 之后

3. **划分执行批次**：

```markdown
## 执行批次

### Batch 0: Foundation
- 任务：基础设施（测试框架、共享库）
- 执行：先行，必须完成
- 并行：否

### Batch 1: Core Features
- 任务：核心功能模块
- 执行：依赖 Batch 0
- 并行：是（同时启动多个）
- 最大并行数：5

### Batch 2: Integration
- 任务：整合层、端到端测试
- 执行：依赖 Batch 1
- 并行：否

### Batch 3: Release
- 任务：文档、发布准备
- 执行：依赖 Batch 2
- 并行：否
```

4. **输出任务清单**：

```markdown
## 任务清单

| Task ID | 任务名 | 文件所有权 | 估计工时 | 依赖批次 |
|---------|--------|-----------|----------|----------|
| T00 | Foundation | lib/common.sh, tests/harness.sh | 1h | Batch 0 |
| T01 | 功能A | bin/a.sh, tests/test-a.sh | 2h | Batch 1 |
| T02 | 功能B | bin/b.sh, tests/test-b.sh | 2h | Batch 1 |
| T03 | 功能C | bin/c.sh, tests/test-c.sh | 2h | Batch 1 |
| T04 | Integration | bin/int.sh, tests/test-int.sh | 3h | Batch 2 |
```

5. **输出文件所有权矩阵**：

```markdown
## 文件所有权矩阵

| 文件 | T00 | T01 | T02 | T03 | T04 |
|------|-----|-----|-----|-----|-----|
| lib/common.sh | ✍️ | 👁️ | 👁️ | 👁️ | 👁️ |
| tests/harness.sh | ✍️ | 👁️ | 👁️ | 👁️ | 👁️ |
| bin/a.sh | 👁️ | ✍️ | ❌ | ❌ | 👁️ |
| bin/b.sh | 👁️ | ❌ | ✍️ | ❌ | 👁️ |
| bin/c.sh | 👁️ | ❌ | ❌ | ✍️ | 👁️ |
| bin/int.sh | 👁️ | ❌ | ❌ | ❌ | ✍️ |
| tests/test-a.sh | ❌ | ✍️ | ❌ | ❌ | ❌ |
| tests/test-b.sh | ❌ | ❌ | ✍️ | ❌ | ❌ |
| tests/test-c.sh | ❌ | ❌ | ❌ | ✍️ | ❌ |
| tests/test-int.sh | ❌ | ❌ | ❌ | ❌ | ✍️ |

图例：
- ✍️ 所有者（可写、可改）
- 👁️ 只读（可读取，不可修改）
- ❌ 禁止（不可访问）
```

### Phase 3: PLAN（规格生成）

**目标**：为每个任务生成详细的 OpenSpec 规格文档

#### OpenSpec 模板

```markdown
# {Task-ID} — {Task Title}

## Execution Order
**{RUN_FIRST | RUN_PARALLEL | RUN_AFTER: [T00, T01, ...]}**

## Branch Isolation Rules

### READ-ONLY Files
以下文件可读取但不可修改：
```
{文件列表，每行一个}
```

### OWNED Files
以下文件本任务完全控制：
```
{文件列表，每行一个}
```

### FORBIDDEN Files
以下文件不可访问：
```
{文件列表，每行一个}
```

## Objective
{一句话描述任务目标}

## Deliverables
- [ ] `{文件路径}` — {功能描述}
- [ ] `{文件路径}` — {功能描述}

## Implementation

### 设计思路
{实现思路说明}

### 代码骨架
```bash
#!/usr/bin/env bash
# {task-id} — {title}

set -euo pipefail

# === 主逻辑 ===
main() {
    # {实现步骤}
}

main "$@"
```

## Test Cases

| ID | Description | Input | Expected Output | Expected Exit Code |
|----|-------------|-------|-----------------|-------------------|
| T1 | {测试场景} | {输入} | {输出} | 0 |
| T2 | {边界测试} | {输入} | {输出} | 0 |
| T3 | {异常测试} | {输入} | {错误信息} | 1 |

## Acceptance Criteria

```bash
# 1. 运行测试
./tests/test-{module}.sh

# 2. 代码检查
shellcheck bin/{module}

# 3. 手动验收
{具体验收命令}
```

## Definition of Done
- [ ] 所有测试用例通过
- [ ] 代码通过静态检查
- [ ] 符合文件所有权规则
- [ ] 已完成自测
```

#### 步骤

为每个任务（T00, T01, ...）按模板生成 OpenSpec 文档。

---

## 完成标志

所有输出完成：
- ✅ 功能点清单
- ✅ 任务清单
- ✅ 依赖图/批次划分
- ✅ 文件所有权矩阵
- ✅ 所有任务的 OpenSpec 文档

**等待用户确认后调用 VK-Execute**

---

## 用户确认点

输出完成后，向用户展示：
1. 任务清单摘要
2. 执行批次顺序
3. 预计总工时

请求确认：
```
规划完成，共 {N} 个任务，预计工时 {X} 小时。
执行批次：Foundation → Core(并行) → Integration → Release

请确认是否开始执行？
- 确认：调用 VK-Execute
- 调整：修改任务划分
- 取消：终止流程
```

---

## 配置选项

可在执行前通过配置调整：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| execution_mode | A/B/C（自动程度） | B |
| max_parallel | 最大并行任务数 | 5 |
| skip_phases | 跳过的阶段 | [] |
| custom_template | 自定义 OpenSpec 模板 | null |
